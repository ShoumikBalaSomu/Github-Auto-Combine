This guide will walk you through the absolute best way to deploy this repository using this free hybrid architecture.

## Step 1: Clone & Hide (The Private Fork - 100% Free)

If you have premium provider URLs, you **cannot** use a public GitHub repository, or your provider will ban your account for leaking credentials. 

1. **Do not** use GitHub's standard "Fork" button (it forces the fork to be public).
2. Create a new **Private Repository** on your GitHub account. *(GitHub allows unlimited private repositories for free!)* Name it whatever you want.
3. Open your terminal or command prompt and run the following commands to copy this project into your new private repo:

```bash
# Clone the original repository to your computer
git clone https://github.com/shoumikbalasomu/Github-Auto-Combine.git my-iptv-project
cd my-iptv-project

# Remove the link to the original repository
git remote remove origin

# Link it to YOUR new private repository
git remote add origin https://github.com/YOUR_USERNAME/YOUR_PRIVATE_REPO.git

# Push the code
git push -u origin main
```

Your code is now completely private and safe.

---

## Step 2: Inject Your Premium Providers (GitHub Secrets - 100% Free)

We don't want to save your premium `username` and `password` inside the code. We will inject them securely via GitHub Secrets.

1. Go to your private repository on GitHub.
2. Click **Settings** ➔ **Secrets and variables** ➔ **Actions**.
3. Click **New repository secret**.
4. Name the secret exactly: **`SECRET_PLAYLISTS`**
5. Paste your premium Xtream Codes M3U URLs (one per line). 
   *Example:*
   ```text
   http://premium-provider.com:8080/get.php?username=YOUR_USER&password=YOUR_PASS&type=m3u_plus&output=ts
   http://backup-provider.com:8080/get.php?username=USER2&password=PASS2&type=m3u_plus
   ```
6. Click **Add secret**.

## Step 3: Enable the GitHub Action (2,000 Free Minutes/Month)

GitHub Actions handles the heavy lifting (scraping, merging, dead-link checking). Even running twice a day, you will only use about 300 minutes per month, well below your 2,000 free minutes!

1. Go to your repository on GitHub.
2. Click the **Actions** tab at the top.
3. You may see a warning saying "Workflows aren't being run on this forked repository". Click the big green button to **Enable Workflows**.
4. On the left side, click **IPTV Playlist Auto-Combine**.
5. On the right side, click **Run workflow** ➔ **Run workflow**.

Wait about 5–10 minutes. The V8 Titan Engine will extract all your live channels, compile the VODs, and build the optimized databases.

---

## Step 4: The Serverless Deployment (Vercel Hobby Tier - 100% Free)

Now we deploy the Xtream API Serverless backend. Vercel's free "Hobby" tier allows you to host APIs and websites completely for free.

1. Go to [Vercel.com](https://vercel.com) and create an account using your GitHub login.
2. Click **Add New** ➔ **Project**.
3. Click **Import** next to your private repository.
4. Leave all the Build & Development Settings exactly as they are.
5. Click **Deploy**.

Vercel will instantly map the `/api/` folder to its Edge Network. It will also host the `/docs/` folder as a beautiful Netflix-style web player.

*Note your Vercel Domain (e.g., `https://my-iptv-project.vercel.app`).*

---

## Step 5: Connecting to Client Apps (OTT Navigator / TiviMate)

Your service is now live! You can share your Vercel URL with your family or users. They don't need M3U links; they can log in like a premium service.

Tell them to configure their IPTV App (like OTT Navigator or IPTV Smarters) exactly like this:

- **Playlist Type:** `Xtream Codes API`
- **Server URL:** `https://my-iptv-project.vercel.app` *(Your Vercel URL)*
- **Username:** `admin` *(or whatever you want, the API accepts anything!)*
- **Password:** `admin` *(or whatever you want)*

### What happens behind the scenes?
1. OTT Navigator asks your Vercel domain for `action=get_live_categories`.
2. Vercel's free Edge Network instantly serves the pre-compiled JSON generated by your GitHub Action in under 50 milliseconds.
3. When the user clicks a channel, Vercel routes them directly to the premium provider source securely.

## Troubleshooting

### Q: Why are my movies and series not showing up?
Ensure you correctly added the `SECRET_PLAYLISTS` in Step 2, and that you manually triggered the GitHub Action (Step 3). The VOD extractor requires premium credentials to pull the movie database.

### Q: The Web Player is showing a 404 error!
If you navigate to `https://my-iptv-project.vercel.app/docs/index.html` and it's blank, ensure your GitHub Action has finished running successfully at least once. The Action generates the `movies.json` and `xtream_live.json` files and places them in `docs/output/`.

No. Vercel's Hobby tier does not even require a credit card. GitHub Actions gives you 2,000 free minutes per month for private repositories, and this script barely scratches that limit. You have a massive, automated IPTV pipeline running 100% in the cloud for $0.00.
